# World Data Explorer

Interactive world atlas that visualizes **30 economic, demographic, social and geopolitical indicators** on a WebGL map — with layer blending, historical time travel and hand-curated diplomacy data.

**Live demo → https://grand-world-data-explorer.vercel.app/**

## Features

- 🗺️ **30 map layers** in 6 groups (Demographics, Economy, Social, Governance, Diplomacy, Map Style), colored per country by real data
- 🎚️ **Combine mode** — blend any set of indicators into a single composite score, normalized per layer (percentile clipping + log/linear scaling)
- ⏳ **Historical layers with time slider** — scrub or play through centuries of population, life expectancy and GDP per capita (Our World in Data / Maddison Project)
- 💬 **Hover tooltips with the math shown** — each value explains how it was computed, including the per-layer normalization breakdown in combine mode
- 🌍 **Country sidebar** with 7 tabs: Economy, Government, Diplomacy, Military, Cultures, Religion, History
- ⚔️ **Diplomatic relations on the map** — wars, conflicts, rivalries and alliances drawn from a hand-researched dataset (Supabase)
- 🚢 **Trade routes layer** — great-circle export/import arcs per country (CIA World Factbook)
- ⚡ Per-indicator caching, debounced layer loading, GPU-evaluated color expressions

## Architecture

100 % client-side SPA — no custom backend. Supabase (Postgres) serves the hand-curated government and diplomacy datasets; everything else comes from public APIs or static JSON.

- **Config-driven layers**: every layer is a declarative entry in [`src/layers.js`](src/layers.js) (source, indicator, unit, scale, format). Data hooks dispatch on the `source` field — adding a World Bank layer is one config object, zero new logic.
- **One MapLibre expression**: each layer resolves to a single `['match', …]` fill expression evaluated on the GPU; hover/selection use imperative `setFeatureState`, so the map never re-renders through React.
- **Reproducible data pipeline**: static datasets in `public/data/` are generated by the scripts in [`scripts/`](scripts/) (`npm run fetch-data`, `npm run fetch-historical`).
- **State**: a single Zustand store ([`src/store/useStore.js`](src/store/useStore.js)) for layers, selection, caches and UI state.

## Running locally

```bash
git clone https://github.com/GabrielLacrampe/world-data-explorer.git
cd world-data-explorer
npm install
npm run dev
```

Government and diplomacy features need a Supabase project; create a `.env` with:

```
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```

All static datasets are committed, so the map works out of the box. To regenerate them: `npm run fetch-data` / `npm run fetch-historical`.
